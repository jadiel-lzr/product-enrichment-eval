<objective>
Implement image pre-flight validation, download caching, and manifest generation for all product images.

Purpose: Cached images enable LLM vision adapters (Claude, Gemini) in Phase 2 to enrich products with multi-modal input. The manifest tracks which images are available so adapters know what they can send. Updating _has_images/_image_count in base.csv creates the A/B grouping (text-only vs text+image) for enrichment comparison.
Output: `data/images/` directory with cached images, `data/image-manifest.json`, and updated `data/base.csv` with accurate image metadata.
</objective>

  <behavior>
    Preflight:
    - checkImageUrl(reachableUrl) returns { url, status: 'reachable', contentType, fileSize }
    - checkImageUrl(unreachableUrl) retries once, then returns { url, status: 'unreachable', error }
    - checkImageUrl(url returning 405 on HEAD) falls back to GET with AbortController, returns reachable
    - runPreflight(urls, concurrency) processes all URLs with p-limit concurrency control

    Downloader:
    - downloadImage(url, destPath) streams image to disk using node:stream/promises pipeline
    - downloadImage(unreachableUrl, destPath) throws with descriptive error
    - downloadAllImages(entries, imagesDir, concurrency) downloads all reachable entries, skips unreachable
    - File extension extracted from URL pathname (strips query params)
    - Files named as {sku}_{index}.{ext}

    Manifest:
    - writeManifest(entries, filePath) writes JSON array with 2-space indent
    - readManifest(filePath) returns parsed ImageManifestEntry[]
    - ImageManifestEntry has: url, status, contentType?, fileSize?, localPath?, error?
  </behavior>
  <action>
    1. Create `enrichment/src/images/manifest.ts`:
       - Export `ImageManifestEntry` type: `{ url: string, sku: string, index: number, status: 'reachable' | 'unreachable', contentType?: string, fileSize?: number, localPath?: string, error?: string }`
       - Export `ImageManifest` type as `ImageManifestEntry[]`
       - Export `writeManifest(entries: ImageManifest, filePath: string): void` -- JSON.stringify with 2-space indent, mkdirSync for directory
       - Export `readManifest(filePath: string): ImageManifest` -- read + JSON.parse

    2. Create `enrichment/src/images/preflight.ts`:
       - Export `ImageStatus` type: `{ url: string, status: 'reachable' | 'unreachable', contentType?: string, fileSize?: number, error?: string }`
       - Export `checkImageUrl(url: string): Promise<ImageStatus>`:
         a. Send HEAD request with 10s timeout (AbortController + setTimeout)
         b. If 200-299: return reachable with contentType and fileSize from headers
         c. If 405: fall back to GET with AbortController, read headers only, abort body, return reachable
         d. If other error: retry ONCE after 1s delay
         e. If retry fails: return unreachable with error message
       - Export `runPreflight(urls: Array<{ url: string, sku: string, index: number }>, concurrency?: number): Promise<ImageManifestEntry[]>`:
         a. Use `p-limit` with concurrency (default 10)
         b. Map each URL through checkImageUrl
         c. Log progress every 100 URLs: "[Preflight] {N}/{total} checked..."
         d. Return array of ImageManifestEntry (without localPath yet -- set after download)

    3. Create `enrichment/src/images/downloader.ts`:
       - Export `extractExtension(url: string): string` -- uses `new URL(url).pathname` to extract extension, defaults to `.jpg` if none found
       - Export `downloadImage(url: string, destPath: string): Promise<void>`:
         a. `fetch(url)` then stream response body to disk via `pipeline(Readable.fromWeb(response.body), createWriteStream(destPath))`
         b. If response not ok, throw with HTTP status
         c. If no response body, throw with descriptive message
       - Export `downloadAllImages(entries: ImageManifestEntry[], imagesDir: string, concurrency?: number): Promise<ImageManifestEntry[]>`:
         a. Filter to reachable entries only
         b. Use `p-limit` with concurrency (default 10)
         c. For each entry: compute destPath as `{imagesDir}/{sku}_{index}.{ext}`, call downloadImage, update entry with localPath
         d. If individual download fails: log warning, set entry status to 'unreachable' with error, continue
         e. Log progress every 50 downloads: "[Download] {N}/{total} saved..."
         f. Ensure `imagesDir` exists with `mkdirSync(imagesDir, { recursive: true })`
         g. Return updated manifest entries (with localPath for successful downloads)

    4. Write tests:
       - `manifest.test.ts`: Test round-trip write/read of manifest, test structure validation
       - `preflight.test.ts`: Mock fetch to test HEAD success, HEAD 405 -> GET fallback, retry on failure, timeout handling. Use vitest mock for fetch.
       - `downloader.test.ts`: Test extractExtension with various URLs (with/without query params, no extension). Mock fetch + stream for download test. Test downloadAllImages skips unreachable.

    5. Run: `cd enrichment && npx vitest run src/images/__tests__/ --reporter=verbose`
  </action>
