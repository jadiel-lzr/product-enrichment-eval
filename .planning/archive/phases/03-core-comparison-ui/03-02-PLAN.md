  <action>
    1. Create frontend/src/hooks/useUrlParams.ts:
       - Uses react-router-dom's useSearchParams
       - Reads URL params: `product` (SKU), `brand`, `category`, `department`, `search`
       - Returns `{ urlSku, urlFilters, setUrlSku, setUrlFilters }` — these are the URL-sourced values
       - `setUrlSku(sku: string)`: updates `?product={sku}` without full navigation (replace: true to avoid polluting history on every click; use push only when the product changes meaningfully)
       - `setUrlFilters(filters: FilterState)`: updates brand/category/department/search params, removes empty params
       - On initial load, returns URL values or null/empty defaults

    2. Update frontend/src/context/ProductContext.tsx:
       - Wire URL params into context: on mount, if URL has `?product=SKU`, use that as initial selectedSku. If URL has filter params, use those as initial filters.
       - When selectedSku changes (via setSelectedSku), also update URL via setUrlSku
       - When filters change, also update URL via setUrlFilters
       - This makes URL the source of truth for navigation state
       - Ensure auto-select logic works: if URL has no product param, auto-select first filtered product AND update URL
       - Back/forward navigation: useEffect watching URL params to sync back into context state

    3. Update frontend/src/App.tsx:

       - Show error state with message if loading fails
       - Wrap in BrowserRouter (if not already from Plan 01)
       - Responsive: on mobile (<768px), sidebar becomes a bottom sheet (Claude's discretion on implementation — can be a slide-up drawer triggered by a floating button, or a full-screen overlay with a "back to comparison" close button). On tablet (768-1024px), sidebar becomes a hamburger-triggered drawer overlay.
