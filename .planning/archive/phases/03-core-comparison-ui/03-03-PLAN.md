}

  <action>
    1. Create frontend/src/lib/field-diff.ts:
       - `getFieldStatus(fieldName: string, enrichedValue: string | undefined, originalValue: string | undefined): FieldStatus`
         - Returns 'enriched' if enrichedValue is non-empty AND differs from originalValue (or originalValue was empty)
         - Returns 'unchanged' if enrichedValue equals originalValue and both non-empty
         - Returns 'missing' if enrichedValue is empty/undefined and the field is a core enrichment target
       - `getFieldColor(status: FieldStatus): { border: string, bg: string, text: string }`
         - 'enriched': border-green-500, bg-green-50, text-green-700
         - 'unchanged': border-gray-300, bg-gray-50, text-gray-600
         - 'missing': border-amber-400, bg-amber-50, text-amber-700
       - Pure functions, no side effects.

    2. Create frontend/src/components/comparison/StatusBadge.tsx:
       - Props: `{ status: 'success' | 'partial' | 'failed' }`
       - Renders a small pill badge: green for success, amber for partial, red for failed
       - Shows text: "Success", "Partial", "Failed"

    3. Create frontend/src/components/comparison/AccuracyScore.tsx:
       - Props: `{ score?: number }` (0-100 scale from LLM self-scoring)
       - If no score, render nothing (null)
       - Renders a small badge/pill showing the score as percentage: e.g., "87%"
       - Color coding: green-600 for 80+, amber-600 for 50-79, red-600 for <50
       - Tooltip or small label "LLM Score" next to the number

    4. Create frontend/src/components/comparison/FieldRow.tsx:
       - Props: `{ fieldName: string, label: string, enrichedValue?: string, originalValue?: string, status: FieldStatus }`
       - Renders a row with colored left border (4px) based on status
       - Left side: field label in small gray text
       - Right side: the value (enriched value if present, otherwise original)
       - Background color from getFieldColor
       - For 'description_eng' field specifically: truncate to ~100 characters with "Show more" / "Show less" toggle button that expands inline. Use local useState for expansion state.
       - For 'missing' status: show italic "Not enriched" placeholder text in amber

    5. Create frontend/src/components/comparison/OriginalDataSection.tsx:
       - Props: `{ product: Product }`
       - Collapsible section (collapsed by default) showing the pre-enrichment original field values
       - Shows the 6 core enrichment field values from the base product as a reference baseline
       - Header: "Original Data" with expand/collapse chevron
       - Uses gray-100 background to visually distinguish from enrichment cards

    6. Create frontend/src/components/comparison/EnrichmentCard.tsx:
       - Props: `{ enrichment: ToolEnrichment, product: Product }`
       - Card layout: white background, gray-200 border, rounded-lg, shadow-sm
       - Card header: tool display name (from TOOL_DISPLAY_NAMES), StatusBadge, AccuracyScore, fields filled count ("4/6 fields" or dynamic N/M based on actual field count)
       - Card body: list of FieldRow components for each CORE_ENRICHMENT_FIELD. For each field, compute status using getFieldStatus comparing enrichment.enrichedValues[field] vs enrichment.originalValues[field].
       - If enrichment has additional fields beyond CORE_ENRICHMENT_FIELDS (keys in enrichedValues not in core set): show expandable "Additional Fields" section below core fields.
       - Failed state: if enrichment.status === 'failed', dim the entire card (opacity-50), show error message prominently, still maintain grid slot (don't remove from grid).
       - Card min-height to prevent layout shift when cards have different content heights.

    7. Create frontend/src/components/comparison/ProductHeader.tsx:
       - Props: `{ product: Product }`
       - Shared header above the 2x2 grid (not repeated per card)
       - Shows: product name (large), brand (medium gray), SKU (small mono), category + department tags
       - Contains OriginalDataSection (collapsible)
       - No image in header (image moves to separate component in Task 2)

    8. Create frontend/src/components/comparison/ComparisonView.tsx:
       - Uses useProductContext to get selectedSku and enrichmentsByProduct
       - If no selectedSku: show helpful empty state
       - Finds the selected product and its enrichments
       - Layout: ProductHeader at top, then 2x2 grid of EnrichmentCard components below
       - Grid: CSS grid with `grid-template-columns: repeat(2, 1fr)` and gap-4
       - For each of the 4 tools (TOOL_NAMES): if enrichment exists for that tool, render EnrichmentCard. If not, render an empty slot with "No data from {tool}" message in gray, maintaining the grid position.
       - Adaptive grid: if only 1-3 tools have data, still show the expected grid positions (Claude=top-left, Gemini=top-right, FireCrawl=bottom-left, Perplexity=bottom-right) with empty placeholders for missing tools.
