    - Test 6 (runner): runBatch writes enriched CSV with original product data + enriched fields + metadata columns

    - Test 8 (runner): runBatch logs progress (e.g., "[claude] 50/498 products processed")

    - Test 10 (report): generateRunReport includes per-field fill rates
    - Test 11 (report): writeRunReport writes JSON report to disk

  <action>
    1. Create `enrichment/src/batch/runner.ts`:
       - Export `RunBatchOptions` interface: `{ adapter: EnrichmentAdapter; products: readonly Product[]; outputPath: string; checkpointDir: string; manifestPath: string; imagesDir: string; concurrency: number }`
       - Export `runBatch(options: RunBatchOptions): Promise<RunBatchResult>` function:
         a. Load checkpoint from `{checkpointDir}/checkpoint-{adapter.name}.json`
         b. Get completed SKUs set from checkpoint
         c. Filter products to only those not yet completed
         d. Log: "[{tool}] Resuming: {completed} already done, {remaining} remaining" (or "Starting: {total} products" if no checkpoint)
         e. Create p-limit instance with `options.concurrency`
         f. Process remaining products concurrently:
            - For each product: load images via `prepareProductImages(product.sku, manifestPath, imagesDir)`
            - Call `adapter.enrich(product, images)`
            - Build enrichment metadata via `buildEnrichmentMetadata(adapter.name, result)`
            - Merge: `{ ...product, ...result.fields, ...metadata }` (immutable spread)
            - Append to checkpoint (atomic write after each product)
            - Log progress every 10 products: "[{tool}] {n}/{total} products processed"
         g. After all products processed:
            - Combine previously-completed products (from checkpoint on resume) with newly-processed products
            - Write enriched CSV to `outputPath` via `writeProductCSV()`
            - Return `RunBatchResult` with all results for report generation
       - Default concurrency per RESEARCH recommendation: Claude=3, Gemini=5, FireCrawl=2, Perplexity=3
       - Export `DEFAULT_CONCURRENCY: Record<string, number>` map

    2. Create `enrichment/src/batch/report.ts`:
       - Export `RunReport` interface: `{ tool: string; totalProducts: number; success: number; partial: number; failed: number; averageFillRate: number; fieldFillRates: Record<string, number>; duration: string; errors: ReadonlyArray<{ sku: string; error: string }> }`
       - Export `generateRunReport(tool: string, results: ReadonlyArray<{ sku: string; result: EnrichmentResult }>, durationMs: number): RunReport`
         - Compute counts by status
         - Compute average fill rate across all products
         - Compute per-field fill rates (for each of the 9 target fields: what percentage of products have that field filled)
         - Collect errors from failed products
       - Export `writeRunReport(report: RunReport, outputPath: string): void` -- writes JSON to disk

    3. Test with mocked adapter (simple mock that returns predefined EnrichmentResult), mocked checkpoint module, mocked CSV writer, mocked image resizer. Verify checkpoint resume logic, progress logging, and report generation.
