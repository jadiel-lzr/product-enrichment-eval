    - Test 4: enrich() falls back to Google Shopping search when brand site search returns no results
    - Test 5: enrich() scrapes top search result as markdown and extracts enrichment fields

  <action>
    1. Install SDK: `cd enrichment && npm install @mendable/firecrawl-js`

    2. Create `enrichment/src/adapters/firecrawl-adapter.ts`:
       - Export `createFirecrawlAdapter(serpApiUrlsPath?: string): EnrichmentAdapter` factory function
       - Initialize `new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY })` client
       - On creation, attempt to load `serpApiUrlsPath ?? 'data/serpapi-urls.json'` as a Map<string, string[]> (SKU -> URLs). If file doesn't exist, log warning and proceed with empty map. This is a soft dependency per CONTEXT.md.
       - `enrich(product, images?)` implementation (images parameter ignored -- FireCrawl doesn't use product images):
         a. Check SerpAPI URLs map for product.sku:
            - If found: skip search, scrape the first URL directly via `firecrawl.scrapeUrl(url, { formats: ['markdown'] })`
            - If not found: proceed to search
         b. Search strategy (per CONTEXT.md: brand site first, Google Shopping fallback):
            - Primary search: `firecrawl.search('{brand} {name} {model}', { limit: 3, scrapeOptions: { formats: ['markdown'] } })`
            - If no results or results empty: fallback search `firecrawl.search('{brand} {name} site:shopping.google.com', { limit: 3, scrapeOptions: { formats: ['markdown'] } })`
         c. Extract enrichment fields from markdown content:
            - Parse the markdown text looking for product details (description, season, collection, GTIN/barcode, dimensions, materials, made_in, weight, year)
            - Use regex patterns + heuristics to extract field values from semi-structured markdown
            - Export a `parseMarkdownForFields(markdown: string, product: Product): Partial<EnrichedFields>` helper (testable separately)
         d. Validate extracted fields with `EnrichedFieldsSchema.parse()`
         e. Compute fillRate via `computeFillRate()`
         f. Return EnrichmentResult (NO accuracyScore -- FireCrawl is not an LLM)
       - Wrap search/scrape calls in `withRetry()`
       - On catch: return failed result

    3. Test with mocked `@mendable/firecrawl-js` (vi.mock). Verify search queries, SerpAPI URL bypass, fallback logic, and markdown parsing.
