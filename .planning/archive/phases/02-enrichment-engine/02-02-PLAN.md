    - Test 5: enrich() uses output_config with zodOutputFormat for structured output
    - Test 6: enrich() returns EnrichmentResult with correct fillRate, enrichedFields, and accuracyScore on success

  <action>
    1. Install SDK: `cd enrichment && npm install @anthropic-ai/sdk`

    2. Create `enrichment/src/adapters/claude-adapter.ts`:
       - Export `createClaudeAdapter(): EnrichmentAdapter` factory function
       - Initialize `new Anthropic()` client (reads ANTHROPIC_API_KEY from env automatically)
       - Model configurable via `process.env.CLAUDE_MODEL ?? 'claude-haiku-4-5-20250415'` (Haiku for cost efficiency per research, configurable for quality testing)
       - `enrich(product, images?)` implementation:
         a. Build prompt text via `buildEnrichmentPrompt(product)`
         b. Build content blocks array:
            - If images provided: map each to `{ type: 'image', source: { type: 'base64', media_type: image.mimeType, data: image.data.toString('base64') } }`, then append `{ type: 'text', text: promptText }` (images before text per Anthropic best practice)
            - If no images: just `[{ type: 'text', text: promptText }]`
         c. Call `client.messages.create()` with `output_config: { format: zodOutputFormat(EnrichedFieldsSchema) }`, `max_tokens: 2048`
         d. Parse response: extract structured output from response content blocks (look for tool_use block with parsed JSON, or text block with JSON)
         e. Validate with `EnrichedFieldsSchema.parse()`
         f. Extract accuracy_score from parsed fields
         g. Compute fillRate via `computeFillRate(parsedFields)`
         h. Return `EnrichmentResult` with status: all 9 filled = 'success', some filled = 'partial', none = 'failed'
       - Wrap the API call in `withRetry()` for automatic 2x backoff
       - On catch: return `{ fields: {}, status: 'failed', fillRate: 0, enrichedFields: [], error: errorMessage }`

    3. Test with mocked `@anthropic-ai/sdk` (vi.mock). Verify content block structure, output_config usage, and result mapping.

    NOTE: Use `messages.create()` (not `messages.parse()`) because `messages.parse()` may not be available in all SDK versions. Instead, manually validate the response with `EnrichedFieldsSchema.parse()` after extracting JSON from the structured output response. The `output_config` field ensures Claude returns valid JSON -- we just need to parse it from the response content.
