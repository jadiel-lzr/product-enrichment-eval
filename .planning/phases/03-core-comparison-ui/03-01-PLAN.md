```typescript
  year: z.string().optional(),
  collection: z.string().optional(),
  gtin: z.string().optional(),

}).strict()
export type EnrichedFields = z.infer<typeof EnrichedFieldsSchema>
```

CSV column convention for enriched CSVs (from Phase 1 research):
- All base columns from Product schema
- Plus: _enriched_fields, _enrichment_tool, _enrichment_status, _enrichment_error
- Plus: _accuracy_score (LLM self-scored, numeric 0-100 or empty)
- Enriched field values overwrite the base column values

       ```typescript
       export const CORE_ENRICHMENT_FIELDS = ['description_eng', 'season', 'year', 'collection', 'gtin', 'dimensions'] as const

         error?: string
         accuracyScore?: number // LLM self-scored 0-100
         fieldsEnriched: number
         totalFields: number
         enrichedValues: Record<string, string> // field name -> enriched value
         originalValues: Record<string, string> // field name -> original value from base.csv
       ```

    1. Create frontend/src/lib/csv-loader.ts:
       - `loadCSV(path: string): Promise<Record<string, string>[]>` — fetches CSV from public/ and parses with PapaParse (header: true, skipEmptyLines: true, dynamicTyping: false — keep all as strings for consistency with Zod coercion)
       - `loadProductCSV(): Promise<Product[]>` — loads public/data/base.csv, parses JSON columns (images, gtin, sizes, errors — same fields as enrichment pipeline), validates each row with ProductSchema (import from @shared/). Collect row-level errors without aborting.
       - `loadEnrichedCSV(tool: ToolName): Promise<ToolEnrichment[]>` — loads public/data/enriched-{tool}.csv. For each row: extract sku, _enrichment_tool, _enrichment_status, _enrichment_error, _accuracy_score. Compare enriched field values against the CORE_ENRICHMENT_FIELDS to determine which were filled. Build ToolEnrichment object with enrichedValues and originalValues. Gracefully return empty array if the CSV 404s (tool not yet run).
       - `loadAllData(): Promise<{ products: Product[], enrichments: Map<string, ToolEnrichment[]> }>` — loads base.csv + all 4 enriched CSVs in parallel with Promise.allSettled. The enrichments Map is keyed by SKU, with value being array of ToolEnrichment (one per tool that has data for that product). Handle individual CSV failures gracefully.

    2. Create frontend/src/lib/mock-generator.ts:
       - `generateMockEnrichedCSV(products: Product[], tool: ToolName): string` — generates a mock enriched CSV string for development. For each product: randomly fill 60-90% of enrichment fields with plausible placeholder values (lorem-style descriptions, realistic seasons like "FW24", years like "2024", collection names from brand). Add _enrichment_tool={tool}, _enrichment_status with 85% success / 10% partial / 5% failed distribution, _accuracy_score as random 50-95 for successful rows. Return CSV string.

    3. Create frontend/scripts/generate-mocks.ts (tsx script):
       - Reads data/base.csv via the enrichment pipeline's parseProductCSV

       - Add package.json script: `"generate-mocks": "tsx scripts/generate-mocks.ts"`
       - Uses PapaParse unparse to generate CSV string, writes with fs

    4. Create frontend/src/hooks/useProductData.ts:
       - Custom hook calling loadAllData() on mount
       - Returns `{ products, enrichmentsByProduct, loading, error }`
       - `enrichmentsByProduct` is a Map<string, ToolEnrichment[]> keyed by SKU
       - Uses useState + useEffect pattern
